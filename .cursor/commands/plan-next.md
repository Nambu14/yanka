Follow the global **incremental-workflow** rule and **whyline-project** context.

## Your job this turn: PLAN ONLY

1. Read `IMPLEMENTATION.md` and find the **next step without ✓** (or the step I name in `$ARGUMENTS` if provided).
2. Read the matching rows in `whyline-spec.md` when relevant.
3. Post a **balanced plan** (not a one-liner, not a long essay):
   - Step ID + milestone
   - Problem (why now)
   - Deliverables
   - Behavior + one minimal example
   - Out of scope
   - Files table
   - Verify commands + pass criteria
4. End with: **Reply `approve <step-id>` to implement.**

## Hard stops — do NOT

- Write or edit product code
- Run implementation “while you’re at it”
- Mark `IMPLEMENTATION.md` ✓
- Commit or push
- Treat “continue”, “go”, or “keep moving” as approval

If I said “continue” without approving a prior plan, this command still means **plan first**, not implement.
